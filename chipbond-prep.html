關鍵品質項目：
• Bump 高度均一性（Height Uniformity）
• Bump 直徑 / 間距（Diameter / Pitch）
• 表面粗糙度（Surface Roughness）
• 無橋接（No Bridging）/ 無缺料（No Missing）
• 剪切強度（Shear Strength）
常見製程失效模式：
• Bump 高度不均 → 電鍍電流分布不均
• Bridging → 間距過小或阻劑曝光不足
• Missing Bump → UBM 附著力問題
• 表面氧化 → 儲存/運輸條件不當

STAR 故事對應頎邦職位需求

STAR ① — NPI 類比：從零建立 SOP（對應「新產品開發導入」）
Situation

研究所研究海洋豚（Lagenodelphis hosei）皮膚組織，全球無現有 IHC 染色文獻可參考，等同新產品導入時無製程記錄可用的情況。

Task

在試樣珍貴且不可重複採集的限制下，建立可移交、可重複執行的完整六模組分析流程。

Action

應用 Taguchi L₁₈ 田口法，對六個製程參數（抗體濃度、孵育時間、抗原修復溫度、修復時間、阻斷液濃度、切片厚度）進行同步優化，將試驗次數從 729 次（全因子）縮減至 18 次，並建立含管制標準的書面 SOP。

Result

Throughput +70%、準確率 92% → 99%、五個機構採用此 SOP、後續批次零失敗。

STAR ② — 異常根因分析（對應「客訴處理」）
Situation

連續兩批 IHC 製程失敗，組織型態嚴重劣化，樣本無法重新取得，且無文獻可查詢根因。

Task

在不浪費珍貴樣品的前提下，精確定位根因並防止重複發生。

Action

停止試驗 → 建立魚骨圖列出所有可能因子 → 5-Why 追溯每一路徑 → 確認交互影響的隱性因素（切片刀刃磨耗 + 緩衝液 pH 漂移）→ 導入 CAPA，更新 SOP 新增量測管制點。

Result

後續批次零重複失敗，此 CAPA 成為 SOP 的永久組成部分。

台大 IEEB 研究生學術競賽，需以全英文向評審委員（含外籍學者）展示複雜的多參數研究設計與統計結果。

Result

獲得最高榮譽（Top Honor），評審特別肯定「清晰的方法論展示」與「量化結果的說服力」。

STAR ④ — 跨機構協調（對應「客戶特殊需求 Support」）
Situation

大型海洋哺乳類野外採樣計畫，涵蓋五個機構、超過 100 名成員，各單位有不同的樣品處理標準與時程。

Task

Q4. 什麼是 SPC？如何在生產中應用？
⚡ 顯示你理解統計邏輯，不只是背定義
SPC（Statistical Process Control）是用統計方法即時監控製程是否處於受控狀態的工具。核心概念是區分隨機變異（Common Cause）和特殊原因變異（Special Cause）——只有後者需要立即行動。

在 Bumping 中，典型的 SPC 應用：
• Bump 高度的 X-bar & R chart：監控批次間均值與變異趨勢
• 當某批次數據點超出管制界限（UCL/LCL），立即 Hold lot 並啟動調查
• Cpk 評估：定期計算製程能力，確認 Cpk ≥ 1.33（一般要求）或 ≥ 1.67（車用要求）

我在研究所使用 SAS 進行過多變量統計分析，理解管制圖的統計基礎，可快速遷移至 Bumping 的 SPC 操作。
Q5. 如果你和客戶對問題的根因有不同意見，你怎麼處理？
⚡ 展示客戶溝通成熟度，不要說「一切以客戶為主」
首先，我會確保用數據說話而不是用意見爭論。如果根因有爭議，表示雙方目前的數據都不夠完整或說服力不足。

我的做法：
1. 收集雙方認為有關的數據，共同檢視（避免各說各話）
2. 如果分歧仍存在，提議設計一個可以區分兩種假設的實驗或分析
3. 在數據完整之前，採取最保守的短期圍堵措施（D3），避免客戶再受損
4. 以書面形式回覆，保留完整的分析記錄

根本原則：維護公司技術信譽，不因壓力而接受未經驗證的根因。
