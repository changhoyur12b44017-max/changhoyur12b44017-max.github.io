面試 Q&A（10 題）

Q1. 你沒有半導體或 Bumping 相關背景，為何認為自己適合這個職位？
核心論點：製程工程的邏輯是通用的。
在研究所，我在沒有任何文獻依據的情況下，獨立建立了完整的六模組組織切片分析流程——這與 NPI 的本質高度相同：面對未知、在有限試驗次數內找到可量產的製程窗口。我應用 Taguchi L₁₈ 田口法同時優化六個參數，僅用 18 次實驗取代 729 次全因子實驗，將throughput 提升 70%，準確率從 92% 提升至 99%。這套「系統性設計實驗 → 數據驅動決策 → 建立可移交 SOP」的思路，正是 Bumping 產品工程師在 NPI 導入時所需要的。缺少的是 Bumping 製程的領域知識，我會在最短時間內補足這個部分。
Q2. 請說明你處理過最困難的製程異常，以及如何解決。
研究所期間，連續兩批 IHC 染色製程失敗，組織型態嚴重劣化，且無文獻可查。我強迫自己停下來，用魚骨圖列出所有可能的影響因子，再以 5-Why 逐一追溯根因，最終鎖定兩個交互影響的隱性因素——切片刀刃磨耗與緩衝液 pH 值漂移——兩者均未納入原 SOP 的管制標準。導入 CAPA 並更新 SOP 後，後續批次零重複失敗。

關鍵訊息：這套 RCA → CAPA → SOP 更新流程，等同於頎邦客訴處理的 8D Report 標準流程，可直接遷移應用。
Q3. 如果客戶抱怨 Bump 高度不均勻，你會如何處理？
步驟框架（即使不懂 Bumping 細節，思路是對的）：
1. 現象確認：收集不良品資料（Wafer ID、Lot#、分佈位置）、確認量測方式是否一致
2. 範圍界定：是單一批次/單一機台/跨批次的系統性問題？
3. 根因分析：電鍍電流密度分布、阻劑厚度均一性、UBM 蝕刻速率差異、設備 PM 週期
4. CAPA：針對根因制定矯正預防措施，設定新的管制限
5. 8D Report：書面回覆客戶，包含短期圍堵與長期預防措施
6. 追蹤：SPC 監控後續批次，確認改善效果
Q4. 客戶稽核時，你如何用英文展示製程能力？
在台大就讀期間，我在 NTU IEEB 學生學術競賽中以全英文簡報獲得最高榮譽，同時曾協調五個跨機構（100人以上）的大型野外作業，對於跨組織英文溝通與簡報有實際經驗。面對客戶稽核，我理解核心是：清晰呈現製程管制邏輯（Control Plan → SPC → CAPA閉環），用數據說話，對不確定的問題誠實說「我確認後回覆」而非猜測。英文表達能力已是我的優勢，半導體製程術語則是我目前正在補充的部分。
Q5. 你如何確保 NPI 在期限內完成，同時維持品質？
研究所 NPI 類比經驗：建立六模組 SOP 時，我先做可行性切割（Feasibility Breakdown），確認每個模組的最小可行驗證次數；再用 Taguchi DOE 最大化每次試驗的資訊量，避免一個變數一個變數地改、浪費時間。在跨機構協調中，我習慣建立明確的里程碑與各方責任矩陣，讓進度可視化。

轉移到 Bumping NPI：同樣的邏輯——先確認哪些關鍵參數最影響 Yield，優先驗證，再逐步收斂製程窗口。
Q6. 什麼是 Cpk？如何用它判斷製程是否穩定？
Cpk（Process Capability Index）= min[(USL−μ)/3σ, (μ−LSL)/3σ]

Cpk 同時反映製程的集中度（偏移）和變異（σ）：
  • Cpk ≥ 1.33：製程能力良好（業界一般要求）
  • Cpk ≥ 1.67：車用/高可靠性要求（IATF 16949）
  • Cpk < 1.0：製程能力不足，需改善
我在研究所的統計分析課程和實驗設計中實際操作過類似的製程能力評估（使用 SAS），理解其背後的統計邏輯；Bumping 製程的 Cpk 計算原理相同，僅是量測項目不同（如 Bump 高度、直徑）。
Q7. 頎邦和競爭對手（如 ASE、日月光）有什麼差異？你為什麼選擇頎邦？
頎邦的定位是DDI 封裝的深度專家而非全能 OSAT，這種專精策略在全球 DDI 市場建立了無可取代的領導地位。我選擇頎邦的原因：
1. 產品工程師職位的完整性：同時涵蓋 NPI 導入、客訴、稽核，能快速建立端對端的產品工程能力
2. 技術深度：Bumping 技術持續演進（Cu-Pillar、SiPh），有長期學習空間
3. IATF 16949 車用認證：接觸到要求最嚴格的客戶品質標準，對個人成長有高度價值
4. 新廠擴張：公司處於積極成長期，個人發展機會更多
Q8. 你的最大缺點是什麼？
我目前最大的不足是缺乏半導體製程的直接業界經驗，特別是 Bumping 製程中的電鍍化學、設備原理，以及半導體工廠的 SPC/SOP 文件管理規範。但我有強烈的學習動機，且我的研究訓練讓我習慣從第一原理理解問題——我相信我可以在 3–6 個月內，透過實際操作和向資深工程師請教，建立足夠的業界知識基礎。文件管理和系統性思維是我現在就有的強項，製程知識是我正在積極補充的部分。
Q9. 你如何看待輪班或加班的要求？
頎邦這個職位是日班（08:30–17:30），週休二日，對我來說是理想的工作安排。在研究所期間，我習慣長時間、高強度的工作節奏——野外採樣需要在有限時間窗口完成大量工作，實驗室 run gel 也常有非標準時間的彈性需求。我理解半導體業有時因客戶緊急需求或異常處理需要加班，這是業界常態，我可以接受合理範圍的彈性工時安排。
Q10. 五年後你希望在頎邦達到什麼目標？
短期（1–2年）：熟練掌握 Bumping 製程知識，能獨立主導 NPI 導入和客訴處理，成為客戶稽核的主要技術聯絡窗口。

中期（3–5年）：晉升為資深產品工程師，能跨產品線（Au / Cu-Pillar / WLCSP）管理多個 NPI 案，並參與客製化製程的技術評估，同時協助培訓新進工程師。

長期：如果頎邦持續在矽光子和化合物半導體擴展，我希望能參與這些新技術的 NPI 導入，把在生命科學研究中養成的「在未知領域中快速建立製程」的能力，持續應用在新興技術上。
